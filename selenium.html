Library

SeleniumLibrary 包含瀏覽器基礎使用的關鍵字。行為包含：尋找頁面指定元素，開啟瀏覽器，截圖，執行 Javascript，滑鼠互動，回傳字串等。。。

※ 關鍵字過多時，若有指定要搜尋的關鍵字，建議按 Ctrl + F 直接搜尋關鍵字。

Alert Should Be Present (警示視窗顯示以及處理) text=（警示視窗顯示文字）, action=ACCEPT (確認警示視窗), timeout=None（設定等待警示視窗出現的時間）
  • Alert Should Be Present 您已經登入成功 5s
    • Alert Should Be Present » 關鍵字
    • 您已經登入成功 » 警示視窗顯示文字
    • 警示視窗處理未設置 » 警示視窗處理方式因未設置所以預設為 ACCEPT (確認)，警示視窗出現後處理方式是確認警示視窗。
      若設為 DISMISS（取消），警示視窗出現後處理方式是取消警示視窗。若設為 LEAVE（不處理），警示視窗出現後會維持視窗開啟狀態；視窗不做任何確認或取消動作。
    • 5s » 等待警示視窗出現時間為 5s
  • Alert Should Be Present 您已經登入成功 DISMISS 10s
    • Alert Should Be Present » 關鍵字
    • 您已經登入成功 » 警示視窗顯示文字
    • DISMISS » 警示視窗處理訊息為 DISMISS，警示視窗出現後會取消警示視窗
    • 10s » 等待警示視窗出現時間為 10s
Handle Alert (警示視窗處理) action=ACCEPT (警示視窗處理), timeout=None（設定等待警示視窗出現的時間）

視窗出現後有三種處理方法：確認視窗 (ACCEPT)，取消視窗 (DISMISS) 或是讓視窗維持開啟狀態 (LEAVE)。

  • Handle Alert (確認警示視窗)
    • Handle Alert » 關鍵字
  • Handle Alert DISMISS 10s (等待 10秒讓警示訊息出現後取消警示訊息視窗)
    • Handle Alert » 關鍵字
    • DISMISS » 警示視窗處理訊息為 DISMISS，警示視窗出現後處理方式是取消警示視窗
    • 10s » 等待警示視窗出現時間為 10s
  • ${message}= Handle Alert LEAVE
    • ${message}= » 把警示視窗的訊息存到 message 裡面。儲存箱。變數。
    • Handle Alert » 關鍵字
    • LEAVE » 使用 LEAVE 讓警示視窗維持開啟狀態，警示視窗不做處理

Get All Links (取得頁面所有連結的 ID)

Get All Links 關鍵字所取得的是原始 HTML 檔案裡面連結設定的 ID。

如果連結尚未設置所屬 ID，測試報告結果回傳的會是空值。

使用方法如下：

  • ${allLinks}= Get All Links
    • ${allLinks}= » 設置變數把關鍵字取得的資訊儲存在這個變數裡面
    • Get All Links » 關鍵字

    如果指定的 cookie 名字 (foo) 不存在，這個測試結果會是執行失敗。

Input Password (輸入密碼) locator（元素定位）: 指定元素位置
password (密碼)： 輸入的密碼
clear=True (清除)： 輸入密碼之前清除欄位既有密碼
  • Input Password xpath=/div/div/password password111 True
    • Input Password » 關鍵字
    • xpath=/div/div/password » 元素定位
    • password111 » 輸入欄位的密碼
    • True » 輸入前先清除欄位既有資訊
Input Text (輸入文字) locator（元素定位）: 指定元素位置
text (文字)： 輸入的文字
clear=True (清除)： 輸入文字之前清除欄位既有文字
  • Input Text xpath=/div/div/text user01 False
    • Input Text » 關鍵字
    • xpath=/div/div/user01 » 元素定位
    • user01 » 輸入欄位的密碼
    • False » 輸入前不清除欄位既有資訊

Wait Until Page Contains (等待直到頁面包含某個文字) text (文字): 指定包含的文字
timeout（超時）: 限制等待出現的時間，如果時間到了文字尚未出現，測試顯示失敗
error（錯誤）: 設定在報告中顯示的錯誤訊息
  • Wait Until Page Contains 登入 5s 等待超時
    • Wait Until Page Contains » 關鍵字
    • 登入 » 元素定位
    • 5s » 設置等待元素顯示的時間
    • 等待超時 » 設置測試失敗顯示的訊息
